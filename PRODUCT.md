# Product

## Register

product

## Platform

terminal

（Textual TUI，无 Web/iOS/Android 形态；impeccable 工具链会将该值按"无额外平台规则"处理。）

## Users

主要使用者是作者本人：一名通过实现 MiniAgent 学习 Agent 架构的开发者，会把它当作日常通用助手使用——写文档、记周报、整理资料，而不只是写代码。次要场景是作品展示：界面会被截图、写进博客或简历，需要在静态截图里也经得起细看。

## Product Purpose

MiniAgent 是一个最小可理解的 Agent 复现：主循环、工具、上下文管理等架构概念都以可读的方式落地。UI 的职责是把设计文档中的状态机（queued → committed、流式草稿、工具调用、运行终态）透明地表达出来，让使用者随时知道 agent 在做什么。做成的标准是视觉打磨到位：接近 Claude Code 等现代 CLI 的完成度，同时保持"看得懂"的气质。

## Positioning

Hermes 式通用个人助手的最小复现。不与主流 CLI agent 竞争功能差异化，而是追求通用（文档、周报、代码任务都同样体面）、透明、可理解。

## Brand Personality

现代精致 CLI。三个词：克制、清晰、耐看。参考 Claude Code 的干净文档流与状态表达，以及 Charm 系工具在配色与 Markdown 渲染上的美感。情绪目标：长时间使用不累，截图展示有信心。

## Anti-references

- 仪表盘化的密集面板（k9s 式到处是框线和分区）
- AI 味装饰：渐变紫、emoji 堆砌、过度框线与卡片
- 聊天气泡与整条消息卡片（设计文档已定为文档流）
- 裸奔的 Textual 默认主题与临时硬编码样式

## Design Principles

1. 界面服务于理解：每个视觉元素都在表达状态机的一个状态；装饰不能先于信息。
2. 排版取胜：终端里只有等宽字体一种字号，层级靠字重、颜色、缩进和留白，不靠框线。
3. 用色克制成系统：一个品牌强调色加一套语义色，各自有明确职责，不为好看而加色。
4. 动效只表达状态：流式输出与运行指示之外不做动画，尊重减少动态的习惯。
5. 通用助手气质：不偏向 coding tool 的仪表盘美学，文档、周报、代码任务都同样体面。

## Accessibility & Inclusion

无特殊无障碍等级要求。默认保证正文对比度不低于 4.5:1；除表达状态的运行指示外不使用动画；状态不只靠红绿色相区分，均配有文字标签。
